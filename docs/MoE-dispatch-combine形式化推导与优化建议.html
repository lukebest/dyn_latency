1 模型与记号

符号含义
\(N\)端侧节点数（= EP size）
\(P\)平面 / rail 数
\(B\)单链路带宽（bytes/s）；有效瓶颈速率 \(P\cdot B\)
\(M[i][j]\)源 \(i\) → 目的 \(j\) 字节数（dispatch / combine 流量矩阵）
\(U_i=\sum_j M[i][j]\)源 \(i\) 上行 egress 负载
\(D_j=\sum_i M[i][j]\)目的 \(j\) 下行 ingress 负载（incast 量）
\(d_{prop}, L_{sw}, s\)单跳传播、交换流水线、单 chunk 序列化 \(s=\text{chunk}/B\)
RTT, BDP端到端往返；\(\text{BDP}=B\cdot\text{RTT}\)

拓扑约束：单台非阻塞 crossbar、输出排队；逐 chunk 跨平面条带；全程无损 CBFC（缓存满 → 链路级反压，零丢包）。

源端纪律：

  • Baseline：单 FIFO / 源（HOL 阻塞）
  • Oracle / SHMEM-POP：per-(src,dst) VoQ 隔离；SHMEM-POP 另加接收端信用窗 \(=\) BDP

2 静态时延（与负载无关）

单包穿越空网的固定流水线开销：

静态时延
\[ T_{static} = \underbrace{2s}_{\text{上下行序列化}} + \underbrace{2d_{prop}}_{\text{上下行传播}} + L_{sw} \]

代入 chunk=4 KB、\(B=25\) GB/s：\(2s\approx0.33\,\mu s\)，\(2d_{prop}=0.2\,\mu s\)，\(L_{sw}=0.3\,\mu s\) → \(T_{static}\approx 0.83\,\mu s\)。任何调度均不能优化（除非改 chunk 粒度或物理参数）。

3 关键路径 makespan：incast floor 推导

设系统瓶颈端口承载字节量：

\[ W^\star = \max\!\Big(\max_i U_i,\ \max_j D_j\Big) \]

dispatch 阶段瓶颈为热点 rank 的下行 \(D_{hot}\)；combine 阶段为热点 rank 的上行 \(U_{hot}\)。

命题（makespan 与调度无关）

在 work-conserving 输出排队交换机中，只要瓶颈端口聚合 ingress 速率 \(>\) drain 速率 \(P\cdot B\)（incast 恒成立），该端口 ramp-up 后永不空闲，故：

关键路径 makespan
\[ \boxed{\,T_{slow} = T_{static} + \frac{W^\star}{P\cdot B} + \Delta_{ramp}\,} \]

其中 \(\Delta_{ramp}=O(\text{buffer}/(PB))\) 为首字节到达 + FIFO 排序残差，有界且不随 \(W^\star\) 增长。定义主项为 incast floor：

\[ T_{incast} = \frac{W^\star}{P\cdot B} \]

含义：只要瓶颈链路持续满负荷，makespan 由该链路串行时间决定，与 VoQ/FIFO 调度顺序基本无关——incast 部分不可优化（只能改 \(P,B,W^\star\)）。

3.1 代入 MoE 流量：四条标度律

设每 rank batch \(T\) token、top-\(k\) 路由、单 token-expert \(b\) 字节；热点 rank 实测入向份额 \(h=D_{hot}/(\text{总字节})\)。总字节 \(=N\cdot T\cdot k\cdot b\)，故：

\[ \boxed{\,T_{incast} = \frac{h\cdot N\cdot T\cdot k\cdot b}{P\cdot B}\,} \]
标度含义仿真验证
\(\propto 1/P\)加平面是降 floor 的唯一物理手段§5.4：371→47 µs（×1/8）；网格 \(P\cdot\)floor ≈ 常数
\(\propto N\)单交换机 floor 随 EP 线性恶化§5.5：371 / 1426 / 2572 µs
\(\propto h\)热点份额线性抬高 floor§5.3：随 ρ_h 线性上升
\(b_{comb}=2b_{disp}\)combine 恰为 dispatch 2×网格热力图精确 2×

4 拥塞扩散（per-flow 动态时延）推导

关键路径之外，真正可优化的大块在冷流（非热点 rank 的完成时间）。

4.1 FIFO + CBFC（Baseline）：冷流被顶到 ≈ 热点 makespan

源 \(i\) 单 FIFO 内热/冷 chunk 按注入序交错。热点输出口缓存满 → CBFC 反压 → FIFO 队头热 chunk 阻塞 → 其后所有冷 chunk 一并停住（HOL）。冷流最后完成时间 ≈ 源 FIFO 排空时间 ≈ 受热点反压支配 ≈ \(T_{hot}\)。

冷流自身下界 \(T_{cold}^{floor}=T_{static}+D_{cold}/(PB)\)，膨胀因子：

冷流膨胀因子（Baseline）
\[ \boxed{\ \eta_{cold} = \frac{T_{cold}^{base}}{T_{cold}^{floor}} \approx \frac{D_{hot}}{D_{cold}} = \frac{h}{(1-h)/(N-1)} = \frac{h\,(N-1)}{1-h}\ } \]

关键推论：\(\eta_{cold}\propto N\)——拥塞扩散随规模线性恶化，比 floor 更值得在大规模下治理。

4.2 VoQ + 接收端信用（SHMEM-POP）：扩散归零

per-(src,dst) VoQ 使热点只阻塞自身队列；接收端信用窗把每目的在途量钳在 1×BDP，缓存填不满 → 反压几乎不触发：

\[ T_{cold}^{pop} = T_{static}+\frac{D_{cold}}{P\cdot B},\qquad \eta_{cold}\to 1 \]

5 缓存需求的推导

要让开环 FIFO 也「不触发反压」从而 baseline 贴 floor，交换机热点输出口须吸收所有并发源的在途突发。并发源数 ≈ \(N-1\)，逐 chunk 摊到 \(P\) 平面，每源至多 1×BDP 在途：

Baseline 匹配缓存（贴 floor 所需）
\[ \boxed{\ \text{Buffer}_{match}(N,P) \approx \text{BDP}\cdot\left\lceil\frac{N-1}{P}\right\rceil \ \propto\ \frac{N}{P}\ } \]

SHMEM-POP 用接收端信用把总在途量钳死，缓存恒为 \(O(\text{BDP})\)：

\[ \text{Buffer}_{pop} = c\cdot\text{BDP}\quad(\text{常数}\approx 98\ \text{KB，与 }N,P\text{ 无关}) \]

仿真：N=128 / P=1 时 \(\text{Buffer}_{match}\approx 3.1\) MB/口 vs POP 98 KB（省 ~32×）。

6 统一形式化结果

单流网络时延分解
\[ \boxed{ \begin{aligned} T_{flow}(f)=\ &\underbrace{T_{static}}_{\text{固定，不可优}} +\underbrace{\frac{W^\star}{P B}}_{\text{incast floor，}\propto\,hN/P\ \text{（不可优，只能改物理）}}\\ &+\underbrace{\mathbb{1}[\text{FIFO,无信用}]\cdot\Big(\frac{W^\star}{P B}-\frac{D_f}{P B}\Big)}_{\text{扩散惩罚，}\propto\,hN/(1-h)\ \text{（可优，VoQ+信用}\to 0\text{）}} \end{aligned}} \]
  • 关键路径（\(f=\) hot）：第三项 = 0，\(T_{slow}\approx T_{static}+W^\star/(PB)\)，调度无关。
  • 冷流（\(f=\) cold）：第三项主导，SHMEM-POP 消除的部分，且随 \(N\) 线性放大。

8 工程优化建议（按公式杠杆排序）

按「作用于哪一项 / 杠杆大小」分三组。\(P,b,h\) 在 floor 上是乘性的——同时下手收益相乘。

A. 压低 incast floor \(W^\star/(PB)\)（唯一能动 makespan 的途径）

杠杆公式依据工程建议量化收益（仿真/推导）
加平面 \(P\) floor \(\propto 1/P\) 最优先。热点 rank 入/出向拆到多平面逐 chunk 条带（multi-rail） P=1→8：371→47 µs（×1/8）
降载荷 \(b\) floor \(\propto b\) dispatch 已 FP8；重点压 combine（2× 大头）：BF16→FP8/低比特、激活压缩 combine 减半 → floor 减半
降热点 \(h\) floor \(\propto h\) aux-loss 均衡、专家副本/热专家多放置、capacity-factor 限流、动态路由 \(h\) 0.45→0.2 ≈ floor 再降 2×
拓扑分层 floor \(\propto N\) 勿把整个 EP 塞进单交换机；Clos/分层，分散热点扇入 N 128→16 同 P 下 floor ×1/8
提带宽 \(B\) floor \(\propto 1/B\) 升级链路速率（次优先，受物理/成本约束） 线性

B. 消除扩散惩罚 \(\propto hN/(1-h)\)（保护冷流/尾延迟）

杠杆公式依据工程建议
源端 VoQ 隔离 消除 \(\mathbb{1}[\text{FIFO}]\) per-destination 队列；热点只阻塞自身 VoQ，杜绝 HOL。把 \(\eta_{cold}\) 从 \(O(N)\) 拉回 1
接收端信用配速 在途量 \(\le\) BDP 防缓存溢出 → 防 CBFC 反压树；保证不触发链路级联回压
不靠深缓存 \(\text{Buffer}_{match}\propto N/P\) N=128 需 ~3.1 MB/口（数百 MB 片上 SRAM，不现实）。把流控从交换机 SRAM 移到端侧信用窗（\(O(\text{BDP})\)）

C. 压低残差 \(\Delta_{ramp}\) 与隐藏硬墙

杠杆依据工程建议
Push/预热 ≤ 1 RTT \(\Delta\) 含首字节 ramp 元数据/信用预交换与 ramp 重叠，gap 钳在 \(O(\text{RTT})\)（仿真 0.84 µs）
计算-通信重叠 floor 是硬墙 专家计算 wave 与通信重叠；端到端由 \(\max(\text{compute},\text{comm})\) 决定
优化目标对齐 makespan 不可优、扩散可优 勿用 makespan 评估调度；应优化 p50/均值/抖动/SM 占用

决策顺序（TL;DR）

  1. 先加平面 + 压 combine 载荷 + 做负载均衡（降 floor，\(P,b,h\) 乘性收益，唯一能动 makespan）。
  2. 再上 VoQ + 接收端信用（消除 \(\propto N\) 的冷流扩散；交换机缓存从 \(O(N/P\cdot\text{BDP})\) 降到 \(O(\text{BDP})\)）。
  3. 用计算重叠藏掉剩余硬墙，并以 p50/抖动而非 makespan 衡量收益。
常见误区：在 node_sweep 图里三方案 makespan 几乎重合，不代表优化无效—— 那只是 incast floor（\(\propto N\)）淹没了 ~25 µs 的 \(\Delta_{ramp}\)。 真实收益在 per-rank 冷流（§5.2：11×→1×）和缓存效率（§5.6：32×）上。
